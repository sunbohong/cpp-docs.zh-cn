# <a name="c26415-smart_ptr_not_needed"></a>C26415 SMART_PTR_NOT_NEEDED

"智能指针参数仅用于访问包含的指针。 请改用 T * 或 T&。 "

**C++ Core Guidelines**： [R. 30](https://github.com/isocpp/CppCoreGuidelines/blob/master/CppCoreGuidelines.md#r30-take-smart-pointers-as-parameters-only-to-explicitly-express-lifetime-semantics)：仅将智能指针作为参数，以显式表达生存期语义

使用智能指针类型将数据传递给函数指示目标函数需要管理所包含的对象的生存期。 但是，如果函数只使用智能指针来访问所包含的对象，并且永远不会调用任何可能导致其解除分配 (的代码，则永远不会影响其生存期) ，通常不需要使用智能指针来使接口变得复杂。 首选对包含的对象的纯指针或引用。

## <a name="remarks"></a>备注

此检查涉及到大多数同时导致 C26410、C26415、C26417 和 C26418 的方案。 更好的做法是先清除 SMART_PTR_NOT_NEEDED，然后切换到共享或唯一指针的边缘事例。 若要进行更集中的清理，可禁用此警告。

除了标准 std：： unqiue_pointer 和 std：： shared_pointer 模板以外，此检查还识别可能会成为智能指针的用户定义类型。 此类类型应定义以下操作：

- 被公开且未标记为已删除的重载取消引用或成员访问运算符。
- 未删除或默认的公共析构函数。 这包括显式定义为空的析构函数。

对可能影响包含对象生存期的操作的解释很广泛，其中包括：

- 接受指向非常量智能指针的指针或引用参数的任何函数
- 复制或移动构造函数或赋值运算符
- 非常量函数

## <a name="examples"></a>示例

繁琐的生存期管理。

```cpp
bool set_initial_message(
            const std::unique_ptr<message> &m) // C26415, also C26410 NO_REF_TO_CONST_UNIQUE_PTR
{
    if (!m || initial_message_)
        return false;

    initial_message_.reset(m.get());
    return true;
}

void pass_message(const message_info &info)
{
    auto m = std::make_unique<message>(info);
    const auto release = set_initial_message(m);
    // ...
    if (release)
        m.release();
}
```

繁琐的生存期管理-改编。

```cpp
void set_initial_message(std::shared_ptr<message> m) noexcept
{
    if (m && !initial_message_)
        initial_message_ = std::move(m);
}

void pass_message(const message_info &info)
{
    auto m = std::make_shared<message>(info);
    set_initial_message(m);
    // ...
}
```
